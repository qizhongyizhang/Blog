写在前面：本文主要记录Hive的基础知识-查询

查询语法

查询顺序

from -> where -> group by -> having -> select -> order by -> limit

查询解释

where

A <=> B：若A、B都为Null，则返回True；任意一方结果部位Null，结果为Null，其他情况与’= ‘操作符结果一致
A rlike B：模糊查询，B为正则表达式，若A与B匹配，则返回True，%匹配任意多个字符，_匹配任意一个字符

sort by：局部排序，仅在Reduce内部进行局部排序

distribute by：类似于MR的partition，进行分区，可结合sort by 使用，需要写在sort by 之前

cluster by： distribute by 和 sort by 是同一个字段时，可用 cluster by 替换，只能使用默认的排序规则倒序，不能指定排序规则

join：表连接时必须进行重命名

  • inner join

on 后面使用的连接条件必须起到唯一键值的作用
inner 可以省略

union：

字段名称必须一致
字段顺序必须一致
没有条件
