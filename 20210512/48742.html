MySQL-安装使用

2021-05-12 安装使用 2.2k 字
数据库 MySQL

写在前面：本文主要对MySQL的安装、卸载和简单配置进行介绍

发展历史

MySQL是由瑞典的MySQL AB公司开发的，目前是Oracle(甲骨文)公司的一个关系型数据库产品(2008年MySQL AB被Sun公司收购、2009年Sun公司又被Oracle收购)，开源免费、扩展方便
MySQL是世界上流行的基于C/S(客户端/服务器端:服务器端：存储和管理数据库的;客户端：发出操作请求的程序)框架的开源的数据库系统
MySQL采用的是C/S架构，需要安装mysqld服务器，mysql客户端，data/数据存储，此外还要配置MySQL服务器占用的资源，使用的连接数，字符集，root用户密码等)

特点

  • 成本低——MySQL代码是开放源的，一般可以免费使用或修改
  • 性能高——MySQL代码执行效率非常快
  • 可信赖——许多厉害的公司、站点在使用和维护MySQL
  • 操作易——MySQL很容易安装和使用

安装卸载

前情

下载官网
完全干净：安装之前确保MySQL相关文件完全清除（第一次安装无此问题），解压下载好的文件至想要安装的目录下(尽量无中文)，检查服务、文件夹和注册表等是否删除完成，环境变量（可以保留）

安装

Windows

  1. 初始化data文件：在bin目录下运行：mysqld –initialize-insecure –user=mysql
  2. 安装MySQL：mysqld -install； 安装完毕，运行配置向导（详细配置：开发环境，服务器环境，独立服务器：占用最多的资源；多功能：支持事务和非事务，支持事务，不支持事务；innodb表空间地址；OLAP，OLTP，手动设置连接数量；网络端口，严格模式；设定字符集；Windows服务名：不允许同名，环境变量；用户登录；运行配置）
  3. 启动服务：net start MySQL
  4. 登录MySQL：mysql -u root -p
    因为之前没设置密码，所以直接回车即可
  5. 设置root用户密码：ALTER USER ‘root’@’localhost’ IDENTIFIED WITH mysql_native_password BY ‘password’;
  6. 刷新权限：flush privileges;
  7. 退出(exit/quit;)重新登陆即可
  8. 配置环境变量：
    • MYSQL_HOME：安装根目录（bin的上一级目录）
  9. 测试
    1. 查看变量：show variable like ‘character_set%’;（%对应任意个数的字符，_ 对应一个字符）
    2. 查看版本：select version();

Linux

  1. 安装之前先查看是否安装mysql：rpm -qa | grep mysql，若有则进入2，若没有则进入3
  2. 卸载mysql，rpm -e mysql-libs，如提示会破坏依赖，则强制删除：rpm -e –nodeps mysql-libs
  3. 安装
  • 安装编译代码包gcc：yum -y install make gcc-c++ cmake bison-devel ncurses-devel
  • 下载安装包
  • 解压安装包：tar -zxvf YUM_package_name
  • 进入解压目录：cd YUM_package_name
  • 编译：
cmake -DCMAKE_INSTALL_PREFIX=/usr/local/mysql
-DMYSQL_DATADIR=/usr/local/mysql/data -DSYSCONFDIR=/etc
-DWITH_MYISAM_STORAGE_ENGINE=1 -DWITH_INNOBASE_STORAGE_ENGINE=1
-DWITH_MEMORY_STORAGE_ENGINE=1 -DWITH_READLINE=1
-DMYSQL_UNIX_ADDR=/var/lib/mysql/mysql.sock -DMYSQL_TCP_PORT=3306
-DENABLED_LOCAL_INFILE=1 -DWITH_PARTITION_STORAGE_ENGINE=1
-DEXTRA_CHARSETS=all -DDEFAULT_CHARSET=utf8
-DDEFAULT_COLLATION=utf8_general_ci
  • 安装：make && make install，时间较长，大概30分钟
  1. 配置：
  • 添加用户和用户组
查看用户列表：cat /etc/passwd
查看用户列表：cat /etc/group
创建组：groupadd mysql
创建用户：useradd -g mysql mysql
修改/usr/local/mysql权限：chown -R mysql:mysql /usr/local/mysql
  • 初始化配置
cd /usr/local/mysql   进入安装目录
scripts/mysql_install_db --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data --user=mysql，配置数据库和数据表存储位置
  1. 启动，在/usr/local/mysql/下运行

注：启动mysql服务时，会按照/etc/my.cnf——“$basedir/my.cnf”搜索mysql的配置文件，新版MySQL的默认位置是：/usr/local/mysql/my.cnf。为防止干扰可将/etc/的文件名修改：mv /etc/my.cnf /etc/my.cnf.bak

  • 复制mysql服务到init.d目录下：cp support-files/mysql.server /etc/init.d/mysql
  • 设置mysql服务开机自启动：chkconfig mysql on
  • 启动mysql：service mysql start
  1. 登录：./mysql -u root -p，回车即可进入，默认密码为空
  2. 初次使用：
    • 修改root用户密码：set PASSWORD = PASSWORD(‘pwd’);
  3. 退出：exit;
  4. 重新登陆：./mysql -u root -p，此时需要登陆密码，密码为之前设置的pwd

卸载

  • 暂停服务:管理员运行cmd：net stop MySQL，再进行删除
  • 控制面板下卸载MySQL，删除ProgramData下的MySQL数据文件
  • 删除注册表：win + r下输入regedit，进入注册表，进行删除
HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\services\eventlog\Application\MySQL
HKEY_LOCAL_MACHINE\SYSTEM\ControlSet002\services\eventlog\Application\MySQL（可能没有） HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\EventLog\Application

连接启动

  • 连接前要先开启服务
开启服务：net start MySQL; (net start service_name);
关闭服务：net stop MySQL;
重启服务：net restart MySQL;

cmd下操作：配置环境变量，可以在任意路径下运行；没配置环境变量，需要在安装路径下的bin目录下运行，下文全是，服务名默认是 MySQL
  • 登录
mysql [-h host_ip -P port] -u user_name -p [passwaord]

主机：运行 MySQL 服务器的主机名地址，如果连接到本地MySQL服务器，为localhost 或 127.0.0.1
端口：默认端口 3306，用默认可省略；修改端口后，需要指定
用户名：一个合法的用户名
密码：用户的登陆密码
  • 测试
输出数据库列表：show databases;  -- 包含MySQL内部使用的数据库，如mysql、information_schema
切换数据库：use database_name;
输出数据表列表：show tables;
  • 断开连接：exit; /quit; /q;

简单查看

查看所有的数据库：show databases [like ‘xxx’];
查看创建数据库的DDL：show create database db_name;
查看当前数据库的所有表：show tables [like ‘xxx’];
查看创建数据表的DDL：show create table tb_name;
查看指定数据库的所有表：show tables from db_name;
查看指定数据表的属性列：show columns from tb_name;
查看广泛的服务器状态信息：show status;
查看授予用户（所有用户或特定用户）的安全权限：show grants；
查看服务器错误或警告消息：show errors / warnings;

远程连接

grant all privileges on dbname.tbname to 'username'@'loginip' identified by 'password' with grant option;

all privileges：表示所有权限（insert、delete、update、select、create、drop、alter等）
dbname：表示数据库名，\* 表示所有数据库
tbname：表示数据表名，\* 表示所有数据表
username：表示用户名
loginip：表示允许远程IP地址，% 表示所有IP地址
password：表示登录密码，为空表示不需要密码登录
with grant option：表示允许级联授权

设置好了之后需要刷新权限：flush privileges;

基础配置

语法校验规则（sql_mode）

查看sql_mode

全局：select @@global.sql_mode;
当前进程：select @@SESSION.sql_mode;

设置sql_mode：set @@sql_mode = sql_mode选项列表
仅对当前进程有效，想要永久有效，需要修改MySQL的配置文件（Wnidows：my.ini/my_dufault.ini；Linux：my.cnf）

sql_mode选项：

  • ANSI_QUOTES：启用ANSI_QUOTES后，双引号将被解释为识别符，不能再用来引用字符串
  • ERROR_FOR_DIVISION_BY_ZERO：启用ERROR_FOR_DIVISION_BY_ZERO后，在INSERT或UPDATE过程中，如果数据被零除，则产生错误而非警告；如果未给出该模式，那么数据被零除时返回NULL
  • NO_AUTO_CREATE_USER：启用NO_AUTO_CREATE_USER后，禁止GRANT创建密码为空的用户
  • NO_AUTO_VALUE_ON_ZERO：默认设置下，插入0或NULL代表生成下一个自增长值，启用NO_AUTO_VALUE_ON_ZERO后，可以在自增长列插入0值
  • NO_ENGINE_SUBSTITUTION：启用NO_ENGINE_SUBSTITUTION后，如果需要的存储引擎被禁用或未编译，那么抛出错误。不设置此值时，用默认的存储引擎替代，并抛出一个异常
  • NO_ZERO_DATE：启用NO_ZERO_DATE后，在严格模式下，不接受’0000-00-00’做为合法日期，可以使用IGNORE选项插入；在非严格模式下，可以接受’0000-00-00’做为合法日期，但会生成警告
  • NO_ZERO_IN_DATE：启用NO_ZERO_IN_DATE后，在严格模式下，不接受月或日部分为0的日期，可以使用IGNORE选项插入；在非严格模式下，可以接受月或日部分为0的日期，但会生成警告
  • ONLY_FULL_GROUP_BY：启用ONLY_FULL_GROUP_BY后，出现在select语句、HAVING条件和ORDER BY语句中的列，必须是GROUP BY的列或者依赖于GROUP BY列的函数列
  • PIPES_AS_CONCAT：启用PIPES_AS_CONCAT后，将”||”视为字符串的连接操作符而非或运算符，与字符串的拼接函数Concat相功能类似
  • STRICT_TRANS_TABLES：启用STRICT_TRANS_TABLES后，如果一个值不能插入到一个事务表中，则中断当前的操作，对非事务表不做限制
