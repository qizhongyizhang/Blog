Git-安装使用

2021-04-08 安装使用 3.3k 字
Tools

写在前面：本文主要记录Git的安装使用的相关知识

介绍

Git是目前世界上最先进的分布式版本控制系统，用C语言编写

  • 版本控制分为集中式和分布式两种；分布式相比于集中式，安全性更高，Git拥有分支管理功能

    • 集中式（CVS、SVN）：版本库是集中存放在中央服务器中，工作时先需要联网从中央服务器下载最新版本，工作结束后再推送给中央服务器
    • 分布式（Git）：没有中央服务器，每台电脑上都有完整的版本库，多人同时修改文件后只需把各自的修改推送给对方，就可以看到对方修改的内容

分布式安全性更高

下载安装

Windows

  1.  官网 
  2. 安装：
    • 01、双击安装包，同意使用许可声明，选择安装路径，Next，
    • 02、选择安装组件，默认选择，Next
      创建桌面快捷方式，在所有控制台窗口中使用 TrueType 字体和是否每天检查 Git 是否有 Windows 更新
    • 03、选择开始菜单页，默认选择，选择Git文件默认的编辑器，默认选择，Next
    • 04、调整您的PATH环境，默认选择，Next
      Use Git from Git Bash Only：仅从 Git Bash 使用 Git，不会修改 PATH ，不能通过第三方软件使用
      Git from the command line and also from 3rd-party software：从命令行以及第三方软件进行 Git，能够从 Git Bash、命令提示符和 Windows PowerShell 以及在 PATH 中寻找 Git 的任何第三方软件中使用 Git ，推荐
    • 05、选择HTTPS后端传输，默认选择，Next
      使用 OpenSSL 库：服务器证书将使用ca-bundle.crt文件进行验证
      使用本地 Windows 安全通道库：服务器证书将使用 Windows 证书存储验证，允许您使用公司的内部根CA证书)
    • 06、配置行尾符号转换，默认选择，Next
      签出Windows风格，提交Unix风格的行尾：签出文本文件时，Git会将LF转换为CRLF；提交文本文件时，CRLF将转换为LF。对于跨平台项目，这是Windows上的推荐设置（core.autocrlf设置为true）
      按原样签出，提交Unix样式的行尾：签出文本文件时，Git不会执行任何转换。 提交文本文件时，CRLF将转换为LF。对于跨平台项目，这是Unix上的建议设置（core.autocrlf设置为input）
      按原样签出，按原样提交：当签出或提交文本文件时，Git不会执行任何转换。不建议跨平台项目选择此选项（core.autocrlf设置为false）
    • 07、配置终端模拟器以与Git Bash一起使用，默认选择，Next
      使用 MinTTY（MSYS2的默认终端）：Git Bash 将使用 MinTTY 作为终端模拟器，该模拟器具有可调整大小的窗口，非矩形选择和 Unicode 字体。 Windows 控制台程序（如Python）必须通过 winpty 启动才能在 MinTTY 中运行
      使用 Windows的默认控制台窗口： Git 将使用 Windows 的默认控制台窗口（cmd.exe），该窗口可以与 Win32 控制台程序（如交互式 Python 或 node.js）一起使用，但默认的回滚非常有限，需要配置为使用 unicode 字体以正确显示非 ASCII 字符，并且在 Windows 10 之前，其窗口不能自由调整大小，并且只允许矩形文本选择
    • 08、安装进度指示，安装完成

Linux

sudo apt-get install git
或者 sudo apt-get install git-core

名词介绍

  • Working Directory：git 仓库的顶级目录就是一个工作区
  • Repository：版本库，工作区的隐藏目录 .git
  • Stage：暂存区，也写作 Index
  • Star：收藏
  • Fork：复制别人的仓库到自己的仓库
  • Pull Request：拉取请求

操作

配置密钥

  1. 设置用户、邮箱
设置用户：git config --global user.name "用户名"
设置邮箱：git config --global user.email "邮箱"
查看用户：git config user.name
查看邮箱：git config user.email
  1. 生成密钥
生成密钥：ssh-keygen -t rsa
查看密钥：cat ~/.ssh/id_rsa.pub

推拉版本库

  1. 新建目录：
mkdir dir_name
  1. 初始化仓库：
git init
进入目录，将目录变成Git可以管理的仓库，自动创建一个 master 分支；当前目录下会增加一个.git目录，用来跟踪管理版本库
  1. 建立远程仓库关联：推拉之前，需要建立本地仓库和远程仓库之间的联系；同时关联多个远程库时，关联时需要为每一个远程库取名
git remote add d_name/origin git@gitee.com:alexedwardz/resource.git
将本地仓库与 Github、Gitee 等远程仓库连接起来，默认是 origin

git remote -v  ：查看远程库信息
git remote rm d_name：删除远程库，解除本地和远程的绑定关系，并不是物理上删除远程库
  1. 添加修改：
git add f_name
将 f_name 添加到本地仓库的暂存区，git add . 表示添加所有文件和文件夹
git add -f f_name：强制将 f_name 添加到本地仓库的暂存区
  1. 提交修改：
git commit -m '自定义备注内容'
将暂存区内容提交到当前分支
  1. 推送：
git push -u origin master
推送代码到远程仓库的 master 分支，第一次提交需要增加 -u 参数，之后提交可以省略
  1. 拉取：
git pull origin master：拉取有关联远程仓库的 master 分支
git clone git@gitee.com:alexedwardz/resource.git：克隆没有关联的远程仓库到本地，git@ 采用的是 ssh 协议

返回版本

未添加：

git checkout -- f_name
撤销 f_name 文件未添加 git add 的修改；git checkout 用版本库里的版本替换工作区的版本

已添加未提交：

1. git reset HEAD f_name；撤销 f_name 文件已添加未提交 git commit 的修改到工作区
2. git checkout -- f_name；撤销 f_name 文件在工作区的修改添加后的新修改

已提交：

  1. 查看更改日志：
git log [--pretty=oneline --abbrev-commit]
查出对应的对应的版本号和版本描述，参数表示一行显示一个更改；在英文状态下输入 q 可退出 log

git log -1 ：显示最近一个提交
  1. 恢复版本：
git reset --hard 版本号
此时在本地文件夹中会恢复到对应版本，git上还是保留最新版本，通过 HEAD 指针指向当前版本，可以快速返回
当前版本可以用 HEAD 表示，上一个版本是 HEAD^，上两个版本是 HEAD^^，上100个版本是 HEAD~100

已退回：已经退回到历史版本，想要恢复到最版版本

  1. 查看命令日志：
    git reflog
    用来记录每一次命令
  2. 恢复版本：
git reset --hard 版本号

分支管理

创建分支：git branch b_name
切换分支：git checkout b_name 或者 git switch b_name
创建+切换分支：git checkout -b b_name 或者 git switch -c b_name
查看分支：git branch，列出所有分支，当前分支前面会标一个 * 号
合并分支：git merge b_name，合并指定分支 b_name 到当前分支

Git 默认执行”快进式合并”（fast-farward merge），会直接将 Master 分支指向 Develop 分支；使用 –no-ff 参数，执行普通模式合并，创建一个新的commit；普通模式合并后的历史有分支，能看出来曾经做过合并，快进式合并看不出来曾经做过合并

合并出现冲突时，需要手动解决冲突再合并：直接在当前分支修改冲突文件，修改成希望的内容，然后添加、提交
git status：可以显示冲突文件
cat f_name：查看文件，通过 <<<<<<<，=======，>>>>>>> 标记不同分支的内容
git log --graph：可以查看分支的合并情况； git log --graph --pretty=oneline --abbrev-commit

git merge --no-ff -m "merge with no-ff" dev：本次合并要创建一个新的commit，需要加上-m参数，添加 commit描述

删除分支：合并完成后删除分支，则又变成一条分支

git branch -d b_name：删除已合并的分支
git branch -D b_name：强制删除未合并的分支

BUG分支：git stash，把当前工作现场“储藏”起来，等以后恢复现场后继续工作

git stash list ：查看隐藏修改内容，
git stash apply [stash@{0}]：恢复不删除 stash内容，通过 list 索引恢复特定的 stash内容
git stash drop  [stash@{0}]：删除 stash内容
git stash pop   [stash@{0}]：恢复、同时删除 stash内容

多人协作

1. git checkout -b dev origin/dev：在本地创建和远程分支对应的分支，名称最好一致
2. git branch --set-upstream-to=origin/dev dev ：建立本地分支和远程分支的关联
3. git pull：拉去远端仓库的分支内容
4. 修改：自己进行修改的内容
5. git add：添加修改
6. git commit -m "xxx"：提交修改
7. git push origin b_name：将修改推送到远端仓库

整理分支：git rebase，把本地未 push 的分叉提交历史整理成直线

标签管理

添加标签：发布版本前，通常先在版本库中打一个标签，唯一确定打标签时刻的版本，标签也是版本库的一个快照；先切换到需要打标签的分支，默认标签打在最新的 commit 上，创建的标签都只存储在本地，不会自动推送到远程

git tag t_name [版本号]：创建名为 t_name，针对特定的版本添加标签
git tag -a t_name -m "xxx" ：创建名为 t_name，说明为 xxx 的标签

查看标签列表：git tag，标签按字母排序，不是按时间排序
查看标签信息：git show t_name
推送到远端：git push origin t_name：推送 t_name 标签到远端；git push origin –tags：推送所有标签到远端
删除标签

删除未推送到远端的标签：git tag -d t_name
删除以推送到远端的标签：
1. git tag -d t_name；先删除本地标签
2. git push origin :refs/tags/t_name；再删除远程标签

其他

查询状态：git status；显示当前分支，当前工作区的文件提交状态
查看区别：git diff HEAD [– f_name]；工作区和版本库里面 f_name 最新版本的区别
删除文件：git rm f_name；删除 f_name 文件；确认删除可以 add、 commit，取消删除可以 checkout
忽略文件：在 Git 工作区的根目录下创建一个特殊的 .gitignore文件，填写需要忽略的文件名，Git会自动忽略这些文件

git check-ignore -v f_name：检查 ignore规则错误
.*：排除所有.开头的隐藏文件
!f_name：排除指定文件，f_name 满足忽略条件，但要求保留

复制提交：git cherry-pick 版本号，制一个特定的提交到当前分支

自定义

显示颜色：git config –global color.ui true
配置命令别名：git config –global alias.n_name command；也可以直接修改当前仓库的 .git/config文件，将别名配置在 [alias] 后面，删除别名可以直接删除行后保存

git config --global alias.st status ：将 status 指令简写成 st
git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

--global 是针对当前用户起作用；如果不加，只针对当前的仓库起作用
当前用户的 Git 配置文件是用户主目录下的一个隐藏文件.git/config；别名就在[alias]后面，要删除别名，直接把对应的行删掉即可

常见问题

搭建Git服务器

  1. 安装Git
    sudo apt-get install git
  2. 创建用户
    sudo adduser git
  3. 创建证书登录：收集所有需要登录的用户的公钥 id_rsa.pub文件，导入到 /home/git/.ssh/authorized_keys 文件里，一行一个
  4. 初始化仓库：在目标目录下操作，如 /srv
    1. sudo git init --bare p_name.git ：创建一个裸仓库没有工作区，仅为了共享
    2. sudo chown -R git:git p_name.git：将仓库所有者改为之前创建的用户 git
  5. 禁用Shell登录：编辑/etc/passwd文件
    将   git:x:1001:1001:,,,:/home/git:/bin/bash
    改为 git:x:1001:1001:,,,:/home/git:/usr/bin/git-shell
  6. 克隆远程仓库
    git clone git@server:/srv/p_name.git
  7. 即可进行推拉

账户密码类

问题1：密码错误，不再出现输入框

  1. Windows 自带的工具凭据管理器：进入控制面板——>用户账户——>管理Windows凭据，找到对应的账户密码可以进行删除或修改
  2. 清除账户密码重新登陆：git config –system –unset credential.helper

仓库权限类

问题1：远程关联已存在：remote origin already exists

  1. 先删除远程 Git 仓库：git remote rm origin
  2. 再添加远程 Git 仓库：git remote add origin git@gitee.com:alexedwardz/resource.git

日常操作类

问题1：拉取前先提交更新，或者隐藏。 Please commit your changes or stash them before you merge.

  1. 提交
git add .
git commit -m 'xxx'
git pull origin master
  1. 隐藏
git stash
git pull
git stash pop
  1. 覆盖本地修改
git reset --hard
git pull
