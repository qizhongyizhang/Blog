Git-安装使用

发表于 2021-04-08 更新于 2021-11-17 分类于 安装使用 阅读次数： Valine：

写在前面：本文主要记录Git的安装使用的相关知识

介绍

Git是目前世界上最先进的分布式版本控制系统，用C语言编写

  • 版本控制分为集中式和分布式两种；分布式相比于集中式，安全性更高，Git拥有分支管理功能

    • 集中式（CVS、SVN）：版本库是集中存放在中央服务器中，工作时先需要联网从中央服务器下载最新版本，工作结束后再推送给中央服务器
    • 分布式（Git）：没有中央服务器，每台电脑上都有完整的版本库，多人同时修改文件后只需把各自的修改推送给对方，就可以看到对方修改的内容

分布式安全性更高

下载安装

Windows

  1.  官网 
  2. 安装：
    • 01、双击安装包，同意使用许可声明，选择安装路径，Next，
    • 02、选择安装组件，默认选择，Next
      创建桌面快捷方式，在所有控制台窗口中使用 TrueType 字体和是否每天检查 Git 是否有 Windows 更新
    • 03、选择开始菜单页，默认选择，选择Git文件默认的编辑器，默认选择，Next
    • 04、调整您的PATH环境，默认选择，Next
      Use Git from Git Bash Only：仅从 Git Bash 使用 Git，不会修改 PATH ，不能通过第三方软件使用
      Git from the command line and also from 3rd-party software：从命令行以及第三方软件进行 Git，能够从 Git Bash、命令提示符和 Windows PowerShell 以及在 PATH 中寻找 Git 的任何第三方软件中使用 Git ，推荐
    • 05、选择HTTPS后端传输，默认选择，Next
      使用 OpenSSL 库：服务器证书将使用ca-bundle.crt文件进行验证
      使用本地 Windows 安全通道库：服务器证书将使用 Windows 证书存储验证，允许您使用公司的内部根CA证书)
    • 06、配置行尾符号转换，默认选择，Next
      签出Windows风格，提交Unix风格的行尾：签出文本文件时，Git会将LF转换为CRLF；提交文本文件时，CRLF将转换为LF。对于跨平台项目，这是Windows上的推荐设置（core.autocrlf设置为true）
      按原样签出，提交Unix样式的行尾：签出文本文件时，Git不会执行任何转换。 提交文本文件时，CRLF将转换为LF。对于跨平台项目，这是Unix上的建议设置（core.autocrlf设置为input）
      按原样签出，按原样提交：当签出或提交文本文件时，Git不会执行任何转换。不建议跨平台项目选择此选项（core.autocrlf设置为false）
    • 07、配置终端模拟器以与Git Bash一起使用，默认选择，Next
      使用 MinTTY（MSYS2的默认终端）：Git Bash 将使用 MinTTY 作为终端模拟器，该模拟器具有可调整大小的窗口，非矩形选择和 Unicode 字体。 Windows 控制台程序（如Python）必须通过 winpty 启动才能在 MinTTY 中运行
      使用 Windows的默认控制台窗口： Git 将使用 Windows 的默认控制台窗口（cmd.exe），该窗口可以与 Win32 控制台程序（如交互式 Python 或 node.js）一起使用，但默认的回滚非常有限，需要配置为使用 unicode 字体以正确显示非 ASCII 字符，并且在 Windows 10 之前，其窗口不能自由调整大小，并且只允许矩形文本选择
    • 08、安装进度指示，安装完成

  • Stage：暂存区，也写作 Index
  • Star：收藏
  • Fork：复制别人的仓库到自己的仓库

分支管理

创建分支：git branch b_name
查看分支：git branch，列出所有分支，当前分支前面会标一个 *号
切换分支：git checkout b_name 或者 git switch b_name
创建+切换分支：git checkout -b b_name 或者 git switch -c b_name
合并分支：git merge b_name，合并指定分支 b_name 到当前分支

Git 默认执行”快进式合并”（fast-farward merge），会直接将 Master 分支指向 Develop 分支；git merge –no-ff -m ‘xxx’ b_name：使用–no-ff参数，执行正常合并，在 Master 分支上生成一个新节点

删除分支：合并完成后删除分支，则又变成一条分支

隐藏修改：git stash，把当前工作现场“储藏”起来，等以后恢复现场后继续工作

整理分支：git rebase，把本地未 push 的分叉提交历史整理成直线
多人协作

标签管理

添加标签：先切换到需要打标签的分支，默认标签打在最新的 commit 上，创建的标签都只存储在本地，不会自动推送到远程

查看标签列表：git tag，标签按字母排序，不是按时间排序
查看标签信息：git show t_name
推送到远端：git push origin t_name：推送 t_name 标签到远端；git push origin –tags：推送所有标签到远端
删除标签

其他

查询状态：git status；显示当前分支，当前工作区的文件提交状态
查看区别：git diff HEAD [– f_name]；工作区和版本库里面 f_name 最新版本的区别
删除文件：git rm f_name；删除 f_name 文件；确认删除可以 add、 commit，取消删除可以 checkout
查看所有提交：git reflog
忽略文件：在 Git 工作区的根目录下创建一个特殊的 .gitignore文件，填写需要忽略的文件名，Git会自动忽略这些文件

git check-ignore：检查 ignore规则错误
.*：排除所有.开头的隐藏文件
!+f_name：排除指定文件，f_name 满足忽略条件，但要求保留

配置别名：git config –global alias.n_name command；也可以直接修改当前仓库的 .git/config文件，将别名配置在 [alias] 后面，删除别名可以直接删除行后保存

常见问题

搭建Git服务器

  1. 安装Git
  2. 创建用户
  3. 创建证书登录：收集所有需要登录的用户的公钥 id_rsa.pub文件，导入到 /home/git/.ssh/authorized_keys 文件里，一行一个
  4. 初始化仓库：在目标目录下操作，如 /srv
  5. 禁用Shell登录：编辑/etc/passwd文件
  6. 克隆远程仓库
  7. 即可进行推拉

账户密码类

问题1：密码错误，不再出现输入框

  1. Windows 自带的工具凭据管理器：进入控制面板——>用户账户——>管理Windows凭据，找到对应的账户密码可以进行删除或修改
  2. 清除账户密码重新登陆：git config –system –unset credential.helper

仓库权限类

问题1：远程关联已存在：remote origin already exists

  1. 先删除远程 Git 仓库：git remote rm origin
  2. 再添加远程 Git 仓库：git remote add origin git@gitee.com:alexedwardz/resource.git
