Nginx-安装使用

发表于 2021-06-28 更新于 2021-11-08 分类于 安装使用 阅读次数： Valine：

写在前面：本文主要记录Nginx的使用

介绍

Nginx：是一个web服务器和反向代理服务器，用于 HTTP 、 HTTPS 、 SMTP 、 POP3 和 IMAP 协议
反向代理：用户将请求发送到代理服务器，代理服务器将请求转发给内部服务器处理，处理完毕后将结果返回给用户，整个过程中客户不和处理服务器建立连接

负载均衡LB：预防服务器出现高并发(多人同时访问同一个接口)，将用户的访问请求尽量均匀的分配给服务器集群
高可用HA：一般设置主节点与备用节点(互为主备)，当主节点宕机后，备用节点可以及时替换主节点，进行后面的任务

正向代理：代理人明确知道用户的目标是谁，代理人帮忙寻找目标，VPN代理

使用场景

高并发场景的负载均衡

高并发指的是在同一时间点有很多请求访问同一 API 接口或者 Url 地址，会给服务器和硬件环境带来很大压力，通过负载均衡(将请求和数据分摊到多个操作单元进行执行)进行处理

下载安装包
解压安装包
安装依赖：yum install gcc pcre-devel zlib-devel openssi-devel -y, nginx需要进行源码编译安装，需要具备C语言编译环境
配置安装目录： ./configure –prefix=catalogue(安装目录)
编译安装：make && make install
开启服务：进入安装目录下， ./sbin/nginx
Ngiinx 的启动都依赖于配置文件：安装目录下的conf目录下的 nginx.conf 文件

负载均衡

  • 软件负载均衡(常见软件有LVS、Nginx、HAProxy)：软件负载均衡可以分为四层负载均衡和七层负载均衡：四层负载均衡利用在网络层中IP地址端口进行请求的转发，起到转发分配作用；七层负载均衡根据访问用户的HTTP请求头、URL信息将请求转发到特定的主机。LVS为四层负载均衡，Nginx、HAProxy可四可七。Nginx是万级的，通常用来做七层负载；LVS是十万级的，可以组合成：LVS+n个Nginx。
  • 硬件负载均衡(常见硬件有F5、A10)：用一个硬件（类似于交换机）和一个基础网络设备，来实现负载均衡。
  • DNS负载均衡：DNS解析同一个域名可以返回不同的ip，用来实现地理级别的负载均衡

DNS负载均衡是地理级别的，硬件负载均衡对应的是集群级别的，软件负载均衡对应的是机器级别的

对比：

  • 软件负载均衡：优点：便宜而且简单灵活，仅安装软件即可，适用于一般小型企业，或者并发量不高的企业，扩展性较好；缺点：和硬件负载均衡比性能一般，不适用于流量很大的企业没防火墙或者防DDos攻击等安全性功能
  • 硬件负载均衡：优点：1.功能强大，支持全局负载均衡提供全面的复杂均衡算法；2.性能强悍，支持百万以上的并发；3.提供安全功能，例如防火墙，防DDos攻击等；缺点：价格昂贵，扩展能力差
  • DNS负载均衡：优点：1.简单，由DNS服务器处理，无需额外操作；2.就近访问可以减少响应时间，提升访问速度；缺点：1.DNS有缓存而且缓存时间较长，当机房迁移等需要修改DNS配置的时候，用户可能还会访问之前的IP，导致访问失败；2.扩展能力差，由运营商管理控制，不能自己定制或者扩展；3.比较笨，不能区分服务器之间的差异，也不能反映服务器的当前运行状态

负载策略

请求轮询：将请求转发给配置的服务器，默认
增加权重：增加服务器权重，权重越大，分发到的请求数也越多，设置权重：weight=n ，默认为1
最少连接数：将请求分发给连接请求数最少的服务器
IPHash：确保来自同一客户端的请求始终定向到同一台服务器
