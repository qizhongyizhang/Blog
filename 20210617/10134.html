IDEA-常见问题

发表于 2021-06-17 更新于 2021-11-08 分类于 常见问题 阅读次数： Valine：

写在前面：本文主要记录 Java 实用工具-IDEA 的常见问题

项目结构类

问题1：cannot resolve symbol “xxxx”，无法解析某方法

原因：

  1. 方法名错误
  2. 方法对应的jar未导入

解决：

针对原因1，检查与定义的方法名是否一致，修改至一致即可
针对原因2，导入jar包（重点介绍）

步骤：先前往官方网站下载对应jar包（注意版本一致）；再将下载的jar导入项目

Plan A：通过 Libraries 添加（推荐）

  1. 打开 Project Structure
  2. 删除自动导入的包

Plan B：通过 Modules 添加（推荐）

  1. 打开Project Structure
  2. 选择导入jar包类型
  3. 导入jar包

Plan C：直接复制：将硬盘上的jar包复制粘贴到项目的lib目录下，不适用于IDEA导入的eclipse项目

外部接口类

问题1：Error:(4, 21) java: 程序包javax.servlet不存在，项目配置中缺少软件包javax.servlet

解决：加入Tomcat
步骤：点击File->Project Structure->Modules;选择当前出错的项目（单击项目名）;选择JDK，点击右侧“+”号，选择“Library…”;选择Tomcat，点击Add Selected按钮，点击Apply按钮，点击OK按钮。

问题2：Error:(10, 37) java: 程序包org.apache.commons.fileupload不存在，项目配置中缺少软件包 org.apache.commons.fileupload

解决：手动下载org.apache.commons.fileupload并安装在Tomcat中
步骤：

  1. 前往http://commons.apache.org/proper/commons-fileupload/download_fileupload.cgi 下载commons-fileupload-1.4-bin.zip文件，将解压得到的文件夹复制到Tomcat安装目录下(与bin目录同级)；
  2. 将包导入项目中：点击File->Project Structure->Libraries；点击右侧“+”号，选择“Java”；选择刚刚的解压文件夹，确认应用的项目，点击Apply按钮，点击OK按钮。

项目文件类

问题1：File is read-only，文件只允许读，

原因：文件设定或者被动被锁定
解决：

  1. 通常编译好的黄色class文件设定为只读，只能编辑对应的找到蓝色java文件
  2. 依次点击：File -> Make File Writable
