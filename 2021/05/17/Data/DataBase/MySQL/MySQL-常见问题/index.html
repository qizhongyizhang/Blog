MySQL-常见问题

写在前面：本文主要记录MySQL使用中的常见问题

账户连接类

  • 问题1：忘记root密码：（Windows）
  1. 关闭MySQL服务；
  2. 在cmd中切换至mysql\bin目录；
  3. mysqld –skip-grant-tables，回车；
  4. 重新打开一个命令行窗口，切换至mysql\bin目录；
  5. 输入mysql，回车，即可登录；
  6. 修改root密码（update修改user表）：
  7. mysql> use mysql; //切换数据库
  8. mysql> update user set password=password(‘123’) where user=’root’ and host=’localhost’;
  9. mysql> update mysql.user set authentication_string=password(‘123’) where user=’root’ ;mysql 5.7的数据库没有password字段,而是authentication_string字段
  10. mysql> flush privileges; //刷新权限
  • 问题2：1130-Host ‘xxx’ is not allowed to connect to this MySQL server
  • 原因：无远程权限
  • 解决：赋予远程权限：update user set host =’%’where user =’root’;刷新权限：flush privileges;
  • 确认：select Host,User from mysql.user; 保证root对应的Host为%，表示任何主机都可以访问
  • 问题3：ERROR 1044 (42000): Access denied for user ‘‘@’localhost’ to database ‘mysql’
  • 原因：‘‘@’localhost’，登录为匿名账户
  • 解决：
  • Linux：
  • 关闭MySQL：service mysqld stop
  • 屏蔽权限：mysqld_safe –skip-grant-table；屏幕出现Starting demo from …..
  • Windows：

关闭MySQL： net stop mysql
屏蔽权限：mysqld –skip-grant-table；屏幕出现Starting demo from …..
新开一个终端登录：mysql -u root mysql，直接Enter进入
重设密码：UPDATE user SET Password=PASSWORD(‘newpassword’) where USER=’root’;
刷新权限：FLUSH PRIVILEGES;

  • 问题4：ERROR 1364 (HY000): Field ‘ssl_cipher’ doesn’t have a default value insert into mysql.user(Host,User,Password) values(“host”,”username”,password(“password”));
  • 原因：mysql中创建用户时，默认配置严格模式，禁止通过insert的方式直接修改mysql库中的user表进行添加新 - 用户
  • 解决：create user ‘username‘@’host’ identified by ‘password’;（不推荐）
  • 修改配置文件： 将sql-mode的STRICT_TRANS_TABLES参数删除，再重启服务，再次添加用户

查询操作类

  • 问题1：1093 - You can’t specify target table ‘XXX’ for update in FROM clause

原因：在同一语句中，不能先select出同一表中的某些值，再update这个表

指令介绍类

  • 问题1：Like 和 Regexp 的区别

like 不使用通配符时，进行等值匹配，如 name like ‘张三’ 等价于 name = ‘张三’；regexp不使用特殊字符匹配时，进行模糊匹配，如 name regexp ‘张三’ 等价于 name like ‘%张三%’；
