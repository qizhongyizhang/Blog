Hive基础-库表数操作

写在前面：本文主要记录Hive的基础知识-库表数操作

HQL指令

数据库

location：指明数据库在HDFS的存放位置，可通过hive-site.xml文件进行配置
with dbproperties()：使用键值对指明数据库的相关描述信息
describe database extended db_name：查看指定数据库的描述性信息
alter database db_name set dbproperties(‘key’=’value’)：修改键值对信息
desc database extended db_name：查看更多详情
drop database database [cascade]：删除数据库，若数据库里面有表，需要加入cascade参数
使用数据库：use db_name;

数据表

if not exists：在不存在的情况下创建表
external：指明创建外部表：外部表仅存储指向实际数据的路径，删除时仅删除元数据；内部表会移动数据到数据仓库指定的路径进行存储，删除时会删除元数据和数据文件
comment：表或字段的描述性信息，默认不能使用中文
partitioned by()：分区表的分区字段，每一个分区单独存在一个目录下
clustered by()：对于每一个表分文件，Hive可以进一步组织成桶，进行更细粒度的范围划分
sorted by() into N buckets buckets：指明排序字段和排序方式
row format ：指明表数据文件的分隔符
storted as：指明数据表文件的数据存储类型

SEQUENCEFILE：数据需要压缩，二进制文件
TEXTFILE：纯文本，默认
RCFILE
location：指明数据表存放位置

内部表
创建

快速建内部表

create table tb_name1 as select * from tb_name2：tb_name1 和 tb_name2 的表结构和数据都一致
create table tb_name1 like tb_name2：tb_name1 和 tb_name2 的表结构一致，但是tb_name1没有数据

查询表信息：desc [formatted] tb_name; 加入 formatted 参数会更详细
删除表：drop table tb_name;

分区表
将一个大的数据文件按照月、天进行分区，即进行切分成小文件，存放在不同的文件夹中，分区就相当于是把表中的数据按照不同的条件分成了不同的文件夹

查询分区：show partitions tb_name;
添加分区：alter table tb_name add partition(col_name=’value1’);
删除分区：alter table tb_name drop partition(col_name=’value1’);
查询分区：select col_name_list from tb_name where col_name_p = ‘value1’;

数据存储格式

行式存储：相邻的是一条记录的不同字段，行查询效率高，列查询效率低，数据压缩效率低
列式存储：相邻的是不同记录的同一字段，行查询效率低，列查询效率高，数据压缩效率高
**TEXTFILE(行式存储)**：默认的存储格式，数据不进行压缩，空间开销大，可结合 Gzip、Bzip2 压缩算法使用
**SEQUENCEFILE(行式存储)**：
**ORC(列式存储)**：每个 ORC 数据文件由多个 stripe 组成，每个 stripe 250M

indexData：某些列的索引数据
rowData：真正的数据内容
StripFooter：stripe 的元数据信息
加载数据时使用 insert into table 语句
stored as orc tblproperties(“orc.compress”=”none”); 指定数据存储方式为 ORC ， ORC 的压缩方式为 NONE(不进行压缩)

**PARQUET(列式存储)**：parquet 文件以二进制方式存储，文件中包含真正数据和元数据，可自解析， parquet 文件内部是由多个行组组成，每一个行组(row group)由 mapreduce 任务进行处理，加载数据时使用 insert into table 语句
