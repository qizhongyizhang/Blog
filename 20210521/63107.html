Redis-基础

发表于 2021-05-21 更新于 2021-11-08 分类于 基础 阅读次数： Valine：

写在前面：本文主要记录数据仓库基础知识

介绍

Redis：Remote Dictionary Server，KV(Key-Value) 数据库，支持高速缓存服务，由单线程+异步IO(多路I/O复用：Tornado-支持万级并发、Node.js)实现

作用：

高速缓存服务（用户经常访问的数据(热数据)从数据库搬到内存）
实施排行榜
投票点赞
消息队列
数据保存方式
rdb：保存数据，redis-check-rdb –fix dump.rdb
aof：记录执行的命令，redis-check-aof –fix appendonly.aof

使用

命令参考文档

  • 连接

启动服务器：redis-server –prot 6379 –requirepass ‘password’ –appendonly yes >redis.log &

也可以通过配置文件方式启动：redis-server 配置文件
连接客户机：redis-cli -h ip -p port
登录：auth password
测试：ping，返回Pong
读写测试：redis-benchmark

  • 应用

新建键值对：set 键值对 ex 正整数，新建一个键值对，在指定正整数秒之后失效
查看现有键：keys key_name，支持通配符*
获取键对应值：get key_name，支持通配符*
返回键值对剩余存活时间：ttl key_name，支持通配符*
保存数据

save：立即保存
bgsave：后台保存

切换数据库：select n，n的取值为(0,1,2…15)，默认为0
清除数据库：

清除当前数据库：flushdb
清除所有数据库：flushall

查看数据库存储大小：dbsize，返回数据库里面存储多小个键值对

数据类型

  • string：

新建键值对：set 键值对 ex 正整数，新建一个键值对，在指定正整数秒之后失效
查看现有键：keys key_name，支持通配符*
获取键对应值：get key_name，支持通配符*

  • hash表：

新建键值对：

hset key_name value ex n，新建一个键值对，在指定n秒之后失效
hmset key_name value ex n，新建一个多重键值对
获取键对应值：
hmget key_name;
hvals key_name;
hgetall key_name;

  • List列表：

新建数据：

lpush
rpush
取出数据
lpop
rpop
返回数据长度：lrange list_name values

  • Set集合：

新建数据：sadd set_name values

  • 有序集合：

新建数据：zadd

  • 地理位置：

新建数据：geoadd geo_name values
计算距离：geodist geo_name name_1 name_2
获取数据类型：type key_name，支持通配符*

配置主从复制

启动：redis-server –slaveof ip port –masterauth password >redis.log $
查看：info replication
结束：slaveof no one
也可以通过修改配置文件进行配置：redis.ini / redis.conf slaveof/replicateof
