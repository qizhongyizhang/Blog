简介

Linux内核版：内核(kernel)，试运行硬件设备的核心程序

Linux发行版：包含内核组件，Ubuntun、CentOS、Redhat等

VＭware安装CentOS

  • 安装：打开VMware，新建虚拟机，自定义安装CentOS

    6. 配置处理器,选择处理器,默认,也可根据自身需要修改,点击下一步;
    7. 配置虚拟机内存,最少628M,不多于自身主机内存的一半,点击下一步;
    8. 配置网络,选择使用网络地址转换(Nat),点击下一步;
    9. 配置I/O,默认(LSI Logic).点击下一步;
    10. 配置磁盘类型,默认(SCSI),点击下一步;
    11. 选择磁盘,选择创建新的磁盘空间;点击下一步;
    12. 指定磁盘大小,默认20G,可根据自身需要修改,点击下一步;
    13. 指定磁盘文件,默认选择,点击下一步;

    15. 点击完成.

    17. 右键单击创建的虚拟机,进入设置;

    20. 选择Install CentOS,Enter;
    21. 选择系统语言,点击继续;
    22. 设置时间日期,地区选Asia,时区选上海,点击完成;
    23. 软件选择,选择最小化安装,点击完成;
    24. 安装目标位置,选择自动配置分区,点击完成;
    25. 点击开始安装;
    26. 设置root密码,点击完成;
    27. 结束后重启,进入系统后,使用root登录.

配置网络

  • 测试网络: ping www.baidu.com;

  • 输入: ip addr,查看网络配置文件，一般为/etc/sysconfig/network-scripts/###fcfg-ens33;

  • 修改网络配置文件:vi /etc/sysconfig/network-scripts/# ###fcfg-ens33,修改”ONBOOT=yes”，保存退出;

  • 关闭防火墙

    systemctl stop firewalld # 临时关闭防火墙
    systemctl disable firewalld # 禁止开机启动

  • 重启网络服务

    service network restart
    隔天打开若无法连接，可以查看虚拟机相关服务是否打开

网络连接模式

  • 桥连接模式：虚拟机IP与本机在同一个网端，通讯通畅，但是会出现IP冲突的危险

  • NAT连接模式：网络地址转换方式，Linux可以访问外网，不会出现IP冲突

    配置IP

    自动获取：

    系统 -> 首选项 -> 网络连接 -> 编辑 -> 勾选自动连接
    自动获取ip的ip地址会改变
    指定固定IP
    修改/etc/sysconfig/network-scripts/ifcfg-eth0

    ONBOOT = yes   -- 启用
    BOOTPROTO = static  -- 指定以静态方式获取IP
    IPADDR = xxx.xxx.xxx.xxx  -- 指定IP
    GATEWAY = xxx.xxx.xxx.xxx  -- 网关
    DNS1 = xxx.xxx.xxx.xxx     -- DNS和网管保持一致即可

    IPADDR、GATEWAY、DNS1的前面三段必须保持一致
    重启网络：service network restart或重启机器：reboot

  • 独立连接模式：Linux独立一个网络IP

  • 测试：ping www.baidu.com

关机

关机重启前应该将内存的数据同步到磁盘上

  • shutdown [选项] [时间]
    shutdown -h now；立即关机
    shutdown -h 1：一分钟后关机，不输入时间，默认一分钟
    shutdown -r now：立即重启
    shutdown -c ：取消关机或重启操作
  • halt：立即关机
  • rebot：重启系统
  • sync：将内存的数据同步到磁盘上
  • logout：退出远程，图形界面运行无效

目录结构

Linux采用基层是的树状目录结构，在Linux世界里，万物皆是文件，包括鼠标、键盘等

root：根目录，对顶级目录，系统管理员的主目录
/bin：二进制指令文件目录；主要用于主体应用
/boot：存放启动Linux的核心文件，包括连接文件和镜像文件
/dev：设备管理目录，将硬件以文件形式存放
/etc：系统管理需要的配置文件和子目录
/home：普通用户的目录，Linux里每一个用户都有各自的 目录
/lib：系统开机所需最基本的动态链接共享库，类似于Window里面的DDL文件
/proc：是一个虚拟目录，是系统内存的映射，存放系统信息
/opt：安装软件管理目录
/sbin：系统管理员专用的二进制指令文件目录，主要用于系统管理
/srv：service的缩写，存放服务启动之后需要提取的数据
/sys：存放2.6内核新出的文件系统ysfs
/tmp：存放临时文件
/usr：用户应用程序和文件的存放路径；/user/bin：后期安装的软件；/user/sbin 超级用户的一些管理程序
/var：日志文件，存放不停变化的文件

分区

分区方式

mbr分区

最多支持四个主分区
系统只能安装在主分区
扩展分区腰斩一个主分区
MBR最大支持2TB，拥有最好的兼容性

gtp分区

支持无限个主分区（受系统限制，Windows 最多支持128个分区）
最大支持18EB的大容量
Windows7 64位之后支持 gtp

管理

挂载：mount
卸载：umont

硬盘

分类

IDE硬盘：并口，用hdxn进行标识，hd表示IDE硬盘，x代表盘号（a：基本盘、b：基本从属盘、c：辅助主盘、d：辅助从属盘），n表示分区数：1-4表示主分区或扩展分区，5以上是逻辑分区
SCSI硬盘：串口，用sdxn进行标识，sd表示SCSI硬盘，x代表盘号（a：基本盘、b：基本从属盘、c：辅助主盘、d：辅助从属盘），n表示分区数：1-4表示主分区或扩展分区，5以上是逻辑分区

查看硬盘分区挂载情况：lsblk -f
返回分区情况、分区类型、UUID（40位字符串的唯一标识）、挂载目录

查看分区大小：lsblk

实践

  1. 新增硬盘
  2. 重启服务器
  3. 创建分区
    操作：
    fdisk /dev/sdb Enter
    输入n           Enter
    输入p           Enter
    连续两次默认选择  Enter
    输入w           Enter

命令

m：显示命令列表
p：显示磁盘分区
n：新增分区
d：删除分区
w：写入并保存退出

  1. 格式化：mkfs -t ext4 /dev/sdb1（ext4为文件类型，/dev/sdb1为分区地址和分区名称）

  2. 挂载：mount /dev/sdb1 /home/newdisk（mount 设备名称 挂载目录，将/dev/sdb1挂载到/home/newdisk下）

    卸载：umont 设备名称/挂载目录，例：mount /dev/sdb1 mount /home/newdisk，取消挂载/dev/sdb1

  3. 设置自动挂载（永久挂载）：vim /etc/fstab 输入挂载信息，保存退出，执行mount -a，即刻生效

文件

Linux系统里面，万物皆是文件，包括鼠标、键盘等外部设备都用文件形式进行保存管理，每一个 Linux 用户最多可以开启1024个进程，每个进程最多可以打开2048个文件

介绍

所有者：文件的创建者
所在组：文件的所在组，一般是创建者的所在组
其他组：除去所在组的其他全部组

管理

新建文件：touch file_name
删除文件：rm file_name
修改所有者：chown user_name file_name；将file_name的所有者改为user_name

同时修改所有者和所在组：chown user_name:group_name file_name，将file_name的所有者改为user_name、所在组改为group_name
级联修改：chown -R user_name dictionary_name，将dictionary_name目录下的所有文件的所有者改为user_name
修改所有组：chgrp group_name file_name，将file_name的所在组改为group_name
级联修改：chgrp -R group_name dictionary_name，将dictionary_name目录下的所有文件的所在组改为group_name

查看文件信息：ls -ahl；以列表形式将所有文件以方便用户查看的形式返回，会显示文件类型、文件权限、所有者、所在组、最新更新时间、文件大小的信息

用户管理

Linux是一个多用户多任务的操作系统，任何一个用户必须使用合法的账号才能登入系统，每一个用户账号至少属于一个用户组，存储在/home目录下；用户信息位于 /etc/passwd 文件中；用户组信息位于 /etc/group 文件中

  • 用户组：类似于角色，对有共性的多个用户进行统一的管理

    添加：groupadd group_name
    删除：groupdel group_name
    查看：cat /etc/group
    修改：chgrp -R group_name file_name/dir_name；-R 进行递归修改

  • 用户：登入系统进行操作的人员

    添加：useradd [-m -g gorup_name] > user_name； -m 自动创建家目录， -g 指定所属组，不指定所属组时，会自动创建与用户名同名的用户组，也可以指定目录：useradd -d 路径 user_name
    设定密码：passwd user_name，回车输入密码；输入密码后才能登陆
    删除：userdel user_name，会保留用户组，如果需要删除用户组，使用：userdel -r user_name，一般保留用户组
    修改用户登陆的初始目录：useradd -d directory_name user_name

    查询：id user_name，返回uid（用户ID号），gid（所在用户组的ID号）和用户组名
    查看当前用户：whoami/who am i；who 查看当前登录的所有用户和登录信息
    修改用户组：usermod -g gorup_name user_name，修改用户的主组（创建用户时指定的组信息）；usermod -G gorup_name user_name，增加用户的附加组；usermod -s /bin/bash user_name；修改用户登录 Shell（默认为 dash）

    超级用户：root 进行系统维护和管理，能够访问所有资源；以上操作都需要通过 root 用户操作
    切换登录用户：su [-] user_name，su 切换至另一个用户，带有 - 表示同时切换到对应用户的家目录，不指定用户名，切换至 root（不安全）exit：撤销切换指令，高权限用户切换到低权限用户不需要输入密码；sudo 使用另一个用户执行，默认为 root

  • 配置文件

    组配置文件：/etc/group，数据格式：user_name:password:uid:gid:注释行描述:主目录：登录shell
    用户配置文件：/etc/passwd，数据格式：groupname:password:gid:userList(会隐藏起来，不可见)
    口令文件：/etc/shadow，加密文件，存放密码信息，删除用户时不会删除相关记录

权限管理

介绍

作用于文件

r：可读，可查看
w：可以修改文件，如想要删除文件，需要拥有对文件所在目录的w权限
x：可以被执行

作用于目录

r：可读，可进入目录查看
w：可在目录内新建、删除、重命名文件或目录
x：可进入该目录

管理

修改权限：chmod（u：所有者、g：所有组、o：其他组、a：所有人）
+、-、=进行权限变更

chmod u=rwx,g=rw,o=r file_name，设置file_name所有者的权限为读、写、执行，所在组的权限为读、写，其他人的权限为读
chmod o+x file_name，给file_name的所在组增加执行权限
通过数字进行权限变更，权限可用数字表示：r=4、w=2、x=1
chmod 764 file_name，设置file_name所有者的权限为读、写、执行，所在组的权限为读、写，其他人的权限为读（7=4+2+1；6=4+2；5=4+1；4=4；3=2+1；2=2；1=1；0：无权限）

Linux软件安装方式

rpm

rpm（RedHat Package Manager），用于互联网下载包的打包和安装工具，类似于Windows的setup.exe程序
查看

查看全部：rpm -qa
查看指定条件过滤：rpm -qa | grep RPM_packake_name
查看软件包是否安装：rpm -q RPM_packake_name
查看软件包信息：rpm -qi RPM_packake_name
查询安装软件包的文件及存放目录：rpm -ql RPM_packake_name
查询指定文件所属软件包：rpm -qf RPM_file_name

管理

安装：rpm [选项] RPM_packake_name

-i：install，安装
-v：verbose，提示
-h：hash，显示进度条
U盘安装：先挂载CentOS的iso文件，在/media/CentOS/下找到到需要安装的软件rpm包，拷贝到/opt/目录下，在/opt/目录下，执行rpm -ivh RPM_packake_name，安装完成

删除：rpm -e RPM_packake_name

删除时如果报错：removing these packages would break dependencies …表示会破坏这个包的相关依赖，可采用rpm -e –nodeps RPM_packake_name进行强制删除，一般不采用，会导致依赖于该软件包的程序无法使用

yum

yum，是Shell前端软件包管理器，基于rpm包管理器，能顾从指定的服务器自动下载RPM包并安装，可以自动处理依赖关系，一次性安装所有以来的软件包，使用yum时需要保证网络通畅，默认安装最新版本

查看

查看全部：yum list installed
查看指定条件过滤：yum list installed | grep name

管理

安装：yum install 软件名

deb

deb

查看

dpkg -l(查看全部)
dpkg-l|grep name(查看特定)

apt

apt：Advanced Packaging Tool，是一个安装包管理管理工具，常用于 Ubuntu 中，用于软件包的安装、删除和管理，在 /etc/apt/source.list 中制定了官方的软件仓库地址（通常在美国）；当执行 install 时，安装相应的包，下载并更新，写入 source.list 文件；当执行 update 时，程序分析 source.list 文件是否有对应的 Packages/Sources/lelease 列表文件，如果有就下载，存入 /var/lib/apt/lists/ 目录中
修改镜像源

  1. 使用前将美国的软件库地址改为清华的软件仓库镜像源：https://mirrors.tuna.tsinghua.edu.cn/
  2. 更新源：sudo apt-get update

管理

更新源：apt-get update
搜索：apt-cache search package_name
获取包信息：apt-cache show package_name，包括说明、大小、版本等
查看包依赖：apt-cache rdepends package_name
安装：apt-get install package_name
修复安装：apt-get -f install package_name
重装：apt-get install package_name –reinstall
升级：apt-get upgrade package_name
删除：apt-get remove package_name
安装编译环境：apt-get build_dep package_name
下载源码：apt-get source package_name
删除包和配置文件：apt-get remove package_name –purge
升级系统：apt-get dist_upgrade package_name
若提示权限不够，可在指令前面加上sudo或者切换到root目录下运行

例：安装sshd服务
安装：sudo apt-get install openssh-server，默认安装服务器端和客户端
启动服务：service sshd start
本机测试：netstat -anp | more
Windows测试：telnet ip port
Linux测试：ssh user_name@ip，退出用exit 或 logout

文本编辑器

vi/vim（vi的增强版）

  • 正常模式：一般模式，使用vi/vim打开即可进入的默认模式，使用上下左右移动光标，使用删除字符和删除整行（dd）、复制、粘贴进行修改文件，不可进行输入
  • 插入模式：编辑模式，在一般模式下按 i、insert 进入编辑模式，可进行正常输入修改
  • 命令行模式：在一般模式下，输入“：或者/”进入命令行模式，输入命令，完成读取、存盘、替换、离开、显示行号等操作
  • 模式切换：vi/vim打开进入一般模式：一般模式下，按 i 或 insert 进入编辑模式，按 Esc 退回一般模式；一般模式下，输入“：或者/”进入命令行模式，按 Esc 退回一般模式；在命令行模式输入 :wq (保存修改退出)、:q (未修改退出)、:q! (不保存修改退出) vi/vim 文本编辑
  • 快捷键

    前往首行：gg，前往尾行：G，正常模式下
    快速前往某行：先设置行号，再输入希望定位到的行号，再按 shift + g，正常模式下
    复制当前行：yy（复制当前后面5行：5yy），正常模式下操作
    粘贴：p，正常模式下操作
    删除当前行：dd（复制当前后面5行：5dd），正常模式下操作
    撤销：u，正常模式下操作
    查找：/查找内容，命令行模式下，按n查找下一个
    设置/取消行号：:set nu/nonu，命令行模式下
